Viua VM is a register-based virtual machine, programmable in high-level assembly; it is designed for reliability, predictability, error-tolerance, concurrency and fine-grained control over behaviour of the software.

The VM achieves its design goals by equipping programmers with a wide array of tools to craft their software.

Reliability and error-tolerance is achieved by a robust system of exception-based error handling, and process restarting.

Predictability and fine-grained control over software is achieved by separating programmers from the machine with only a thin layer of programming language (a high-level assembly).

The "synchronised" and "unsynchronised" are used instead of "asychronous" and "sychronous" on purpose.

VM supports two approaches to concurrency: an unsychronised, actor-based model using processes communicating via asynchronous message-passing; and a synchronised model using coroutines communicating via calls and return values.

Features

Machine provides programmers with the following set of features ("WIP" means "Work In Progress", "PF" means "Planned Feature"):

  • compile-time checks of code perfomed by the assembler
  • separate compilation (assembler is able to produce "object files")
  • static and dynamic linking
  • hot-swapping dynamically linked modules (WIP)
  • exception-based error handling (every object is throwable and catchable; VM catches C++ exceptions thrown in foreign functions)
  • first-class functions
  • arity-based function overloading
  • mechanisms for static and dynamic dispatch of method calls
  • concurrency
  • parallelism (given sufficient hardware resources)
  • scope-based resource management (no garbage collector)
  • coroutines (PF)
  • non-blocking foreign function interface (requires at least two-core CPU)
  • virtual process restarting (via user-programmed watchdog process)
  • clean FFI for straightforward cooperation with C++ code
  • no-copy function returns
  • by-copy and by-move parameter passing to functions
  • safe pointers for inside-VM objects
  • move semantics for many instructions
  • closures with multiple capture mechanisms (by-copy, by-reference, by-move)

Requirements

Machine requires the following software to be available:

  • a modern C++ compiler supporting C++14; builds are tested with GCC, and Clang++
  • Python 3.5 (or later) is required to run the test suite
  • recent version of Valgrind (3.11 and later) is required for memory tests
  • GNU Make (or compatible build system) is required to compile the code
  • developer scripts assume presence of standard *nix tools: sed, dc, find, xargs, a sh-compatible shell, and Linux operating system
  • compilation without developer scripts requires a *nix-like operating system

Also, it is assumed that the machine is compiled and run on a *nix-like operating system, and on C++ runtime with enabled exceptions.

Development status

Viua is late-alpha grade software, it is by no means production ready.
